# 原始方法论：AI Coding 闭环调试 G1 训练方法论

> 来源：sonic_agent_handoff_20260508_stablebody 项目
> 本文档为关键摘录，供迁移参考

---

## 核心理念

AI agent 驱动的闭环研发，核心在于：
1. **自动化反馈循环**：代码变更 → 运行 → 采集反馈 → 分析 → 下一轮变更
2. **视频第一性**：AI 必须直接观察行为，不能只依赖日志和数值
3. **长期记忆**：AGENT.md 积累可复用原则，避免重复犯错
4. **渐进式 PR**：每个 PR 只解决一个问题，有严格门控

## 七大机制

### 机制 1: PR 驱动的渐进式开发
- 将长期研发目标分解为有严格依赖关系的 PR
- 每个 PR 只解决一个核心问题
- 每个 PR 有明确的验收标准和门控条件
- PR 验收时必须确认前一个 PR 的产物仍然可用

### 机制 2: N 层信号分离
- 将问题空间按信号链拆为 N 层
- 每层有独立的分析工具
- 先定位层，再深入诊断
- 核心原则：如果上游信号已经异常，优先查上游

### 机制 3: 视频第一性原则
- AI agent 不能只查代码日志，必须直接"看"视频
- 视频录制自动化：每个评估 episode 自动生成视频
- 多视角录制：全局视角 + 局部视角
- 视频参数：适当分辨率，30fps，H.264

### 机制 4: 多模态 AI 视频审查（子 Agent 隔离）
- 用多模态大模型作为"第二双眼睛"审查视频
- 子 Agent 架构：独立环境，不共享主 Agent 上下文
- 输出结构化 JSON 报告
- 三层证据交叉验证：数值分析 + 视频审查 + 关键帧目视

### 机制 5: 分层分析器矩阵
- 每个信号链层有对应的分析脚本
- 分析器输出数值指标和判定
- 分析器之间可并行运行

### 机制 6: AGENT.md 长期记忆
- 本文件即为长期记忆
- 每轮实验后提炼原则写入
- 只写可复用的原则和判断逻辑
- 不写流水账、不写临时猜测

### 机制 7: Git 精确追溯
- 每个实验 run 记录 git commit
- 记录配置参数和环境信息
- 支持任意实验的回溯和对比

## 闭环数据流

```
用户给出长期研发目标
    │
    ▼
AI Agent 自主执行：
    a. 写代码 / 改配置
    b. 运行评估 episode
       → 启动仿真环境
       → 执行目标任务
       → 录制视频
       → 采集指标
    c. 运行分析器
       → 各层分析脚本
       → 输出数值报告
    d. 视频审查（子 Agent）
       → 多模态 AI 看视频 + 打分
    e. 综合判断
       → 视频目视 + 分析器数值 + 子 Agent 审查
       → 输出：PASS / FAIL / HIGH_RISK
       → 定位问题在第几层
    f. 更新 AGENT.md
       → 将发现转化为可复用原则
    g. 向用户报告 + 准备下一轮
```

## 打分准则设计

- 总分 100 分
- 5-10 项结构化检查
- 每项有：观察要点、评分标准、扣分规则
- 总体判定：PASS (80-100) / HIGH_RISK (60-79) / FAIL (0-59)
- 异常扣分项：摔倒、卡死、崩溃等严重问题

## 关键设计决策

1. **子 Agent 隔离**：视频审查必须在独立环境中运行，确保客观性
2. **三层证据交叉**：数值分析 + 视频审查 + 关键帧目视，三者交叉验证
3. **渐进式 PR**：每个 PR 只解决一个问题，避免大范围变更
4. **长期记忆**：AGENT.md 积累可复用原则，新 session 读取即可继续
5. **精确追溯**：每个实验 run 记录 git commit，支持回溯和对比
